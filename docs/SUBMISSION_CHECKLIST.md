# DoraHacks Submission Checklist

This list tracks what must be ready before final submission.


## 1) Code & Tests

- [x] `forge test` passes locally.

- [x] Fork suite A-F passes (`test/ForkSuite.t.sol`).

- [x] Slither run completed; notes in `docs/SLITHER_NOTES.md`.

- [x] On-chain checks recorded in `docs/ONCHAIN_CHECKS.md`.


## 2) Documentation

- [x] `README.md` is fully English.

- [x] `ARCHITECTURE.md`, `ECONOMICS.md`, `THREAT_MODEL.md` present.

- [x] `docs/ANALYSIS.md` (Part1-3) completed.


## 3) Demo Video (Required)

- [ ] Record a 3-minute demo video.

- [ ] Upload video and capture final link.

- [ ] Verify the demo follows `docs/DEMO_SCRIPT.md`.


## 4) Submission Form (DoraHacks)

- [ ] Project name + one-line pitch.

- [ ] Track / tags selection.

- [ ] GitHub repo link.

- [ ] Demo video link.

- [ ] Team member info.

- [ ] Contract addresses (if required by track).

- [ ] Optional: slide deck link.
